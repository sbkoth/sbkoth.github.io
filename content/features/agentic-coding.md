# Agentic Coding & AI-Native Engineering

Building software with coding agents as first-class collaborators—not as a demo, but as a daily engineering practice. Plans, implementation, tests, and review run in tight loops while a human owns intent, architecture, and merge criteria.

## How it shows up in practice

- **Agent-orchestrated delivery** — Break work into verifiable increments; agents implement, tests prove, humans decide
- **Spec → plan → PR** — Design docs and task checklists drive agent sessions so changes stay scoped and reviewable
- **Verification before “done”** — Typecheck, unit tests, lint, and static build gates must pass before claiming success
- **Tool-using agents** — File edits, shell, browsers, and MCP integrations under explicit capability boundaries
- **Portfolio as proof** — This terminal site itself is maintained with heavy agentic coding on a pure static GitHub Pages path

## Engineering discipline that keeps agents safe

- **Import-graph hygiene** — Delete dead weight; keep only what the live UX path needs
- **Pure logic for tests** — Command dispatch, sanitize, export, and autocomplete stay unit-testable without a browser
- **Sanitized content surfaces** — Markdown rendered only through marked + DOMPurify pipelines
- **Static production model** — No runtime app server in production; content validated with Zod at export time
- **Accessibility & performance** — Live regions, combobox patterns, bundle and asset discipline stay in the loop

## Where this meets platforms & AI products

Agentic coding is complementary to data platforms and GenAI product work: the same systems thinking applies to RAG pipelines, ML platforms, and event-driven backends—just with agents accelerating the path from idea to shipped artifact.
